Variables

There is 3 ways to define a varaible

  • var

    The original and old way to define variables. What needs to be known is var is able to be changed at any time. No block scope, the value can be changed inside a function. Anything that is defined with var will be hoisted will be initialized asundefine.

  • let

    Introduced in ES6/2015. Now has code block meaning that function keeps variables declared in the scope of a function. It can also be redefined as a new value. Is hoisted to the top of the block but is not initialized to a value and the outcome is ReferenceError: Cannot access 'calledBefore' before initialization. which is different from "ReferenceError: 'notInCode' is not defined".

  • const

    Introduced in ES6/2015. Now has code block meaning that function keeps variables declared in the scope of a function. The big difference is that the const is a constant which means that the value can not be changed. Is hoisted to the top of the block but is not initialized to a value and the outcome is ReferenceError: Cannot access 'calledBefore' before initialization. which is different from "ReferenceError: 'notInCode' is not defined".

Function

Functions can take parameters or not, even take multiple parameters. Which are bound to the scope of the function

Functions produce values and will need to return said value. but functions can produce side effects. Like removing a CSS class one a HTML tag

Allowing to extract a section of code into smaller bits. Removes duplication of code and keeping codebase easier to understand. JavaScript has 3 basic ways to define a function

  • Regular Named Declaration

    Pretty standard way of declaring function,

    function logInCaps (item){ const capItem = item.toUpperCase() console.log(capItem) return capItem }

  • Varaible Declaration

    Same capabilities as a regular declaration, but is possible to store a function as a value, pass as an argument, and so on. Basically do the same thing as regular declared values.

    const logInCaps = function (item){ const capItem = item.toUpperCase() console.log(capItem) return capItem }

  • Arrow functions

    ES6/2015 feature. Usesthis from the immediate outer scope, avoiding binding to the scope where declared ... .bind(this). Able to have an implicit return, no need to have a return.

    Not able to use arrow function as a constructor, since the this is bound to the created function

    const logInCaps = (item) =>{ const capItem = item.toUpperCase() console.log(capItem) return capItem }
Scope

Basicly two type of scopes, global and local. Global is able to access variables that are defined by let or const and other bindings from any where in the code base. Local are varaiables that are contained within a fucntion and everytime a function is called a new instance of the variable are created. Variables that are local are only accessable to the function where are defined and any children functions that are defined inside the scope of the partent function.

Hoisting

Is a JavaScript mechanism that will bring place the variables to the top. The JavaScript engine finds which will allow the variables to be used before they are declared. This doesn't mean that the variables are initialized, the only declaration of said variables. This is why we get an 'undefined' if we call a variable before it is initialized to a value.
